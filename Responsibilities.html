RoleDescription
Square
  • Responsible for setting up the repository. This includes naming the repository and adding team members.
  • Ensure everyone has his or her own branch to work from
  • Create a new branch for each topic
  • Keep only tested, working code (or final reports, READMEs, and other documentation) in the master branch so that there is always something functional to present
  • Privacy of the repository should be set to “public.”
  • To create a new branch, run the following command: git branch -b "branchname"
Triangle
  • Responsible for creating a simple machine learning model
  • Create a simple model that isn’t concerned with accuracy. Basically, you want your model to take input data and return a label and that’s it!
  • Questions:
    • Which model did you choose and why?
    • How are you training your model?
    • What is the model’s accuracy?
    • How does this model work?
Circle
  • In charge of the mockup database:
    • SQL-based database, including an ERD
    • A document pointing out:
      • How it is integrated into your database
      • How it works with the code
    • Use either sample data or even fabricated data to test it
    • Submit training and testing data
    • By the date of your virtual class, you’re required to submit the mockup database to your instructional staff.
    • Create a document describing the schema of the database (this can be a markdown document, or an ERD)
X
  • Focus on the technology side of the project. Consider these questions: . Which tools are the best fit for your project? . What will be used for each section? . How will the dashboard be built?
  • By the live session, . submit a list of technologies you and your team intend to use for the project. . explain which technology is being used where
  • Create a document named “technology.md” and push it up to your branch.
